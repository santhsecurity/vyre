# CRITIQUE_CODEX_2026-04-22

Scope reviewed: `libs/performance/matching/vyre/CLAUDE.md`, `libs/performance/matching/vyre/audits/RELEASE_GATE.md`, `libs/tools/surgec/SCOPE.md`, the `surgec` engine, the `vyre` backend/frontend surface, and the current CI gates. Every finding below is line-anchored to code or workflow content that was read directly.

1. 🔴 CRITICAL | `libs/tools/surgec/src/taint.rs:72-88` | `NoOpOracle::successors` and `NoOpOracle::predecessors` always return empty vectors even though the public API still advertises taint reachability, which guarantees silent false negatives for any flow-sensitive rule. Fix: delete the no-op oracle path and replace the trait entrypoint with a real graph-backed taint engine that lowers taint edges into `vyre` programs and is exercised by scan-time evaluation. Cross-ref: `FINDING-SURGEC-4`.
2. 🟠 HIGH | `libs/tools/surgec/src/lib.rs:50-55` | The crate still publicly re-exports the deprecated taint-oracle surface, which keeps a knowingly incomplete abstraction alive in the API even though the scope requires direct graph semantics. Fix: remove the deprecated export, wire the public taint surface to the real v3 graph pipeline, and provide a single supported path through `compile`/`lower`/`scan`. Cross-ref: `FINDING-SURGEC-4`.
3. 🟠 HIGH | `libs/tools/surgec/src/lower/mod.rs:42-49` | `lower_document` silently skips every rule that lacks `report`, so a document can parse and compile while entire rule classes never enter the executable program set. Fix: lower all rule forms into explicit result sinks, or fail compilation when a rule shape cannot yet be represented instead of dropping it.
4. 🟠 HIGH | `libs/tools/surgec/src/lower/mod.rs:145-176` | Literal lowering narrows all integer literals to `u32`, which loses signedness and large-width semantics needed for the arbitrary-compute model described in `SCOPE.md`. Fix: carry integer width and sign in the lowered IR and emit the matching `vyre` constant op instead of truncating to `u32`.
5. 🟠 HIGH | `libs/tools/surgec/src/lower/mod.rs:155-155` | Floating-point literals are truncated from `f64` to `f32` during lowering, so source rules cannot express exact thresholds without precision loss. Fix: preserve `f64` in the lowered representation and add backend support for the wider numeric lane rather than casting on ingest.
6. 🟠 HIGH | `libs/tools/surgec/src/lower/mod.rs:156-176` | String, regex, and identifier literals are reduced to a `u32` hash, which creates collision-driven false positives and false negatives in a security engine that is supposed to preserve exact semantics. Fix: intern literals into a stable string table and lower references by table index plus exact byte payload, not by lossy hash.
7. 🔴 CRITICAL | `libs/tools/surgec/src/lower/mod.rs:219-268` | Comparison lowering builds expressions over `__cmp_lhs` and `__cmp_rhs` but never binds those names in the emitted program, so every lowered comparison reads undefined IR state. Fix: allocate fresh temporary bindings for the left and right subprogram results, insert them explicitly into the combined program, and reference those temporaries in the comparison op.
8. 🔴 CRITICAL | `libs/tools/surgec/src/lower/mod.rs:312-323` | `Expr::IsMember` lowers only the family bitset and never composes it with `inner_prog`, so the membership test described by the AST is not implemented at all. Fix: thread the inner expression result into the emitted membership op and intersect it with the resolved family/zone bitset before reporting a match.
9. 🟠 HIGH | `libs/tools/surgec/src/lower/mod.rs:341-344` | `Expr::Aggregate` hard-errors in the lowerer, leaving all aggregate semantics promised by the language and scope unavailable in the executable path. Fix: introduce explicit aggregate nodes in `vyre` lowering for reductions, accumulators, and grouped outputs, then compile aggregate AST nodes into those ops instead of aborting.
10. 🟠 HIGH | `libs/tools/surgec/src/lower/mod.rs:345-348` | `Expr::Fixpoint` is rejected outright, which means recursive dataflow and convergence rules described by the universal engine scope do not exist in code. Fix: add strongly connected component scheduling and a bounded fixpoint operator to the lowered program model, then compile recursive rule blocks through that operator.
11. 🟠 HIGH | `libs/tools/surgec/src/lower/mod.rs:349-352` | `Expr::Path` still terminates with an error, so path-sensitive reasoning is missing from the core lowerer even though the engine is supposed to express multi-hop exploit chains. Fix: lower path expressions into graph-walk primitives with explicit edge constraints, path length bounds, and path materialization.
12. 🟠 HIGH | `libs/tools/surgec/src/lower/mod.rs:353-356` | `Expr::Motif` is unimplemented in the executable pipeline, which blocks graph motif detection promised by the scope innovations. 13. 🟠 HIGH | `libs/tools/surgec/src/lower/mod.rs:357-360` | `Expr::Comprehension` still hard-errors, so rules cannot synthesize derived collections or candidate sets from prior matches. 14. 🟠 HIGH | `libs/tools/surgec/src/lower/mod.rs:361-373` | Universal/existential quantifiers are not lowered, which means the language cannot express whole-set or bounded-search invariants required for universal detection.
