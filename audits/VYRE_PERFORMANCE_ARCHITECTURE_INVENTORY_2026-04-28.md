# VYRE Performance And Architecture Inventory

Date: 2026-04-28

Status legend: `open` means not fully fixed in the current tree.

## P0 Dispatch And Readback Performance

1. `fixed` Replace per-output readback mutex slots in `vyre-driver-wgpu/src/engine/record_and_readback.rs` with lower-overhead completion channels or a compact shared completion array. Fixed by replacing `Arc<Mutex<Option<_>>>` map slots with bounded callback channels plus atomic readiness flags.
2. `fixed` Make direct readback ring the default hot path instead of allocating/mapping per dispatch. Fixed by adding backend-owned size-classed `ReadbackRingSet` arenas, recording ordinary output copies into reusable ring slots from the main dispatch encoder, arming ring tickets after submission, and gating the default with `scripts/check_direct_readback_ring_default.sh`; trap and timestamp sidecars stay on pooled staging because they require lazy full-sidecar remapping and query payload isolation.
3. `fixed` Add zero-copy output APIs so callers can consume mapped POD slices instead of always receiving `Vec<Vec<u8>>`. Fixed by `WgpuBackend::dispatch_borrowed_for_each_mapped_output` and `dispatch_borrowed_for_each_pod_output`, which visit trimmed mapped readback slices in place and unmap before returning visitor errors.
4. `fixed` Add caller-owned or arena-owned output buffers to remove one heap allocation per output. Fixed by adding `dispatch_borrowed_into` to backend and compiled-pipeline contracts and wiring wgpu pending readback into caller-owned output storage. Extended with `WgpuBackend::dispatch_borrowed_batch_into`, which reuses one caller-owned `OutputBuffers` slot per batch job.
5. `fixed` Finish migration from `&[Vec<u8>]` dispatch APIs to borrowed `&[&[u8]]` across all backend surfaces. Fixed for hot production/conformance callers by routing matching and conform/lens dispatch loops through `dispatch_borrowed`; `scripts/check_no_owned_dispatch_hot_paths.sh` is now part of release signoff so `.dispatch(` cannot re-enter those hot roots. Legacy `dispatch` remains as the compatibility surface for external callers and tests.
6. `fixed` Add typed dispatch surfaces for `u32`, `f32`, byte ranges, and POD structs. Fixed by adding the public `TypedDispatchExt` backend extension trait with `dispatch_bytes`, `dispatch_pod<T: Pod>`, `dispatch_u32`, and `dispatch_f32`, re-exported from `vyre-driver` and `vyre`, plus unit coverage for typed packing and malformed output widths.
7. `fixed` Add multi-program GPU batching so many independent small programs compile/submit/read back as one queue batch. Fixed by splitting wgpu dispatch into record-only and submit phases, recording each borrowed batch job into an owned `RecordedDispatch`, submitting all command buffers through one queue submission for the backend device, and mapping every readback from the shared submission index while preserving per-job result slots and caller-owned output buffers.
8. `fixed` Precompute binding lookup maps per pipeline; avoid linear scans through buffer bindings during record. Fixed by replacing repeated binding linear scans with `BindingLookup`, using inline storage for small programs and a hash index for larger binding sets.
9. `fixed` Move dispatch-local `SmallVec` fallbacks into per-thread scratch arenas for large programs. Fixed by adding per-thread `DispatchScratch` arenas for record/readback clear requests and bind-group cache-key/index scratch, wiring `record_and_submit_async` through the arena, and retaining vector capacity across large-program dispatches without leaking owned GPU buffer lifetimes into the scratch.
10. `fixed` Add allocation-count tests around direct dispatch, async dispatch, and compiled pipeline dispatch. Fixed by `vyre-driver-wgpu/tests/dispatch_allocation_contract.rs`, which measures steady-state heap allocations/bytes for direct, async, and compiled dispatch against ratcheted budgets.
11. `fixed` Add GPU timestamp query instrumentation for upload, dispatch, copy, map, and readback phases. Fixed by opt-in `gpu-timestamps`/`wgpu-timestamps` dispatch profiles: wgpu now enables `TIMESTAMP_QUERY_INSIDE_ENCODERS`, writes query boundaries around compute and readback-copy phases, resolves/maps the timestamp buffer, emits `vyre.wgpu.timestamps` telemetry, and fails loudly if requested profiling lacks required GPU features.
12. `fixed` Replace remaining `Maintain::Wait` hot-path waits with submitted-fence wait only where strict synchronization is required. Fixed by changing backend flush to submit an explicit empty queue fence and wait for that submission index, routing command/pipeline validation through a nonblocking error-scope poll, removing ad hoc pipeline/subgroup `Maintain::Wait` calls, and tightening `scripts/check_no_hot_path_blocking_wait.sh` to the two remaining one-shot device-acquisition waits.
13. `fixed` Add async readback aggregation for independent dispatches so CPU waits once for a group. Fixed by collecting `WgpuPendingReadback` handles through `WgpuPendingDispatch::await_batch_owned`, polling once per device queue for the latest submitted batch item, then harvesting each mapped readback in original job order with per-job error isolation.
14. `fixed` Make trap sidecar readback optional and lazily mapped only when trap state indicates a possible failure. Fixed in `vyre-driver-wgpu/src/engine/record_and_readback.rs`: dispatches without trap tags still omit the sidecar, and trap-enabled dispatches now map only the first 4-byte trap flag on the normal path; the full sidecar is mapped and decoded only when that flag is nonzero.
15. `fixed` Add perf regression thresholds for short-program dispatch latency and long-buffer throughput. Fixed in `vyre-driver-wgpu/tests/dispatch_hot_path.rs` with cached small-dispatch latency, batched-submit latency, and hot 4 MiB long-buffer dispatch/readback throughput budgets.
