# Vyre Release Roadmap

> **v0.4.1 RESTRUCTURING PREFERENCE  -  set 2026-05-02.** v0.4.1 is the first
> public push. Crate names, file paths, and `pub use` shape are what external
> consumers will link against, so churn there has higher cost than usual.
> Until v1, prefer optimization and innovation work over restructuring; if a
> restructure or rename genuinely unblocks a meaningful optimization, ship it
>  -  just be deliberate about whether the win pays for the consumer-side
> churn. Default tilt is "stable surface unless we're moving the needle." No
> hard ban; this is a thumb on the scale, not a freeze.

This is the canonical executable roadmap for optimization, structure, and
performance work. Older roadmaps and audits are evidence. Work is claimed in
`CLAIMS.toml`, placed by `OWNERSHIP.toml`, and proven by the patch contract in
`AGENT_CONTRACT.md`.

The north star is the vision file: Vyre is the missing GPU stratum. Semantics
live once, backends translate, and the megakernel is the default runtime path.
