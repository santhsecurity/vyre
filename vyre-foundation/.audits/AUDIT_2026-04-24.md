## Findings

[CRITICAL] wire/decode/from_wire.rs:137 — `bytes_extraction` flag is hardcoded to `false` during decode; wire round-trip silently resets `BufferDecl::bytes_extraction`. Fix: encode the flag in metadata payload and decode it into `DecodedBuffer`.

[CRITICAL] wire/encode/to_wire.rs:289 — `put_output_set` is a permanent stub that always writes a zero-length set. The OutputSet wire section is specified but unimplemented. Fix: implement output-set serialization or remove the section from the VIR0 spec.

[CRITICAL] optimizer/passes/normalize_atomics.rs:26 — Pass is a complete no-op: `changed = false` with no tree rewrite. Claims to fix M3 `AtomicExchange`-in-conditional quirk but does nothing. Fix: implement the `Let tmp = Atomic(); If(tmp, ...)` rewrite or delete the pass.

[CRITICAL] optimizer.rs:303 — `fingerprint_program` calls `program.to_wire().unwrap_or_default()`; any unserializable program collides to the same fingerprint (empty bytes). Fix: propagate the `Result` or use a distinct failure digest.

[HIGH] ir_inner/model/program/builder.rs:119 — `Program::with_rewritten_entry` omits `non_composable_with_self`, so every optimizer pass that rewrites the entry silently drops the flag. Fix: copy `non_composable_with_self` into the returned `Program`.

[HIGH] optimizer/passes/autotune.rs:42 — `Autotune::transform` calls `Program::wrapped(...)` without preserving `non_composable_with_self`. Fix: chain `with_non_composable_with_self(program.is_non_composable_with_self())`.

[HIGH] optimizer/passes/fusion.rs:30 — `Fusion::transform` copies `entry_op_id` but drops `non_composable_with_self`. Fix: preserve the flag on the rewritten program.

[HIGH] optimizer/passes/dead_buffer_elim.rs:33 — `DeadBufferElim::transform` drops `non_composable_with_self`. Fix: preserve the flag via `with_non_composable_with_self`.

[HIGH] optimizer/passes/const_fold.rs:20 — `ConstFold::transform` drops `non_composable_with_self` via `with_rewritten_entry`. Fix: preserve the flag after `rewrite_program`.

[HIGH] optimizer/passes/strength_reduce.rs:24 — `StrengthReduce::transform` drops `non_composable_with_self` via `rewrite_program`. Fix: preserve the flag on the returned program.

[HIGH] validate/validate.rs:14 — `validate_self_composition` is commented out, disabling self-composition validation. Programs with duplicate self-exclusive regions pass validation silently. Fix: uncomment the import and invoke it in `validate_with_options`.

[HIGH] validate/validate.rs:702 — `visit_trap` only checks depth limits; it does not recurse into `address` with `validate_expr` or reject empty tags. Fix: validate the address expression and tag content.

[HIGH] validate/expr_rules.rs:62 — `Expr::Call` validation recurses into arguments but never checks that `op_id` is non-empty or well-formed. Empty `op_id` strings pass silently. Fix: reject empty or malformed `op_id` values.

[HIGH] wire/decode/from_wire.rs:253 — `workgroup_size` is decoded without validating dimensions are non-zero. A crafted blob with `[0,0,0]` decodes successfully but cannot re-encode. Fix: add a zero-dimension rejection in `from_wire`.

[HIGH] wire/encode/to_wire.rs:77 — `reject_non_roundtrippable_shapes` rejects workgroup_size=0 at encode time, but the decoder has no symmetric check. Fix: mirror the encode-time validation in `from_wire`.

[MEDIUM] validate/typecheck.rs:163 — `BinOp::Opaque(_)` bypasses all operand type validation. Fix: call `validate_extension()` on opaque binops or reject them until a validator is registered.

[MEDIUM] validate/typecheck.rs:360 — Unknown `UnOp` variants in `expr_type` push `None`, which downstream code treats as an unconstrained type. Fix: default to a conservative type or emit a validation error.

[MEDIUM] validate/depth.rs:63 — `max_call_depth` ignores `op_id` and never walks the callee graph; it admits arbitrarily deep call chains. Fix: walk the dialect registry or require pre-inlined programs.

[MEDIUM] validate/nodes.rs:276 — `Node::Trap { .. } | Node::Resume { .. } => {}` in the legacy validator passes through without real checks. Fix: validate trap address and resume tag.

[MEDIUM] optimizer/passes/spec_driven.rs:23 — `SpecDriven` is a no-op pass that returns an unchanged program but is still scheduled in the default pipeline. Fix: remove from built-in list or implement algebraic rewrites.

[MEDIUM] wire/decode/from_wire.rs:432 — `read_output_set` reads the output set count and discards every value without storing them. Fix: persist the output set into `Program` or remove the wire section.

[MEDIUM] wire/encode/to_wire.rs:215 — `output_byte_range` start/end are encoded as `leb_u64`, but `meta.rs:415` canonical equality key truncates them to `put_u32`. Fix: use consistent width in both wire format and canonical key.

[MEDIUM] wire/encode/to_wire.rs:313 — `memory_kind_tag` maps `Persistent=>6, Push=>5`, but `meta.rs:404` canonical key maps `Persistent=>5, Push=>6`. Fix: align canonical key mapping with wire tags.

[MEDIUM] ir_inner/model/program/meta.rs:301 — `compute_wire_hash` panics via `to_wire().expect(...)` on any unserializable program. Fix: return `Result` and let callers handle the error gracefully.

[MEDIUM] optimizer/scheduler.rs:50 — `PassScheduler::default()` panics on invalid pass metadata via `.expect(...)`. Fix: return `Result` or remove the panicking `Default` impl.

[MEDIUM] validate/validate.rs:296 — `self.scope_stack.pop().unwrap()` panics on stack underflow. Fix: use `pop().ok_or(...)?` with a meaningful error.

[MEDIUM] optimizer/scheduler.rs:144 — `.expect("Fix: scheduled pass id must exist in pass index.")` in production scheduler code. Fix: return `OptimizerError` for internal inconsistency.
